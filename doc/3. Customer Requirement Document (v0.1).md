# Customer Requirement Document (v0.1)
*예시가 영어길래.. 소제목은 영어로 적어봤습니다*
## 1. Background
사용자는 일정 관리 과정에서 작업을 반복적으로 미루는 경향이 있으며, 실제 마감 직전에 작업이 집중되는 문제가 발생한다. 기존 일정 관리 서비스는 단순히 기록 기능을 중심으로 하여 사용자의 행동을 실제로 유도하는 데 한계가 있다. 이에 따라, 작업 수행 시점을 앞당기고 사용자의 행동 패턴을 분석할 수 있는 시스템이 필요하다.
## 2. Overall Goals
본 시스템은 다음과 같은 목표를 가진다.
- 사용자의 작업 수행 시점 앞당긴다.
- 가짜 마감 일정을 통해 행동을 유도한다.
- 작업 수행 데이터를 기록하고 분석한다.
- 사용자가 자신의 행동 패턴을 인식할 수 있도록 한다.
## 3. Fake Deadline
- 사용자는 작업과 실제 마감 일정을 등록할 수 있어야 한다.
- 시스템은 실제 마감 이전에 가짜 마감 일정을 자동으로 생성할 수 있어야 한다.
- 가짜 마감은 사용자가 인지할 수 있도록 표시되어야 한다.

## 4. Task Tracking
- 사용자는 작업 시작, 중단, 완료 상태를 기록할 수 있어야 한다.
- 시스템은 작업 수행 시간과 이벤트를 자동으로 저장해야 한다.
- 기록된 데이터는 사용자별로 관리되어야 한다.

## 5. Data Analysis
- 시스템은 사용자의 작업 수행 데이터를 기반으로 분석 결과를 제공해야 한다.
- 분석 항목은 평균 집중 시간, 작업 완료까지 소요된 시간, 시간대별 수행 패턴을 포함해야 한다.
- 분석 결과는 사용자가 이해하기 쉬운 형태로 제공되어야 한다.

## 6. Plan vs Actual Comparison
- 사용자는 계획한 수행 시간과 실제 수행 시간을 비교할 수 있어야 한다.
- 시스템은 이러한 비교를 시각적으로 제공해야 한다.

## 7. Behavioral Pattern Analysis
- 시스템은 반복적인 작업 수행 데이터를 분석할 수 있어야 한다.
- 평균 집중 지속 시간, 작업 시작 후 중단까지 걸리는 시간, 작업별 평균 수행 시간, 일정 시간 내 중단 발생 비율과 같은 지표를 포함해야 한다.

## 8. Data-based Scheduling Support
- 시스템은 과거 데이터를 기반으로 작업 별로 예상 소요 시간을 제공할 수 있어야 한다.
- 사용자는 이를 참고하고 계획을 수정할 수 있어야 한다.

## 9. Notifications
- 시스템은 가짜 마감 도달 시 사용자에게 알림을 제공해야 한다.

## 10. Performance Expectations
- 작업 기록 및 조회는 지연 없이 이루어져야 한다.
- 분석 결과는 사용자 요청이 들어올 경우 빠르게 제공되어야 한다.

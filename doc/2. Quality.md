# 대상 시스템 품질 요소 추정하기
## (1) 사용자가 원하는 기대치는 무엇인가?
### (1)-1. 정확성
- 가짜 마감 생성이 의도한 규칙대로 실제 마감 이전의 적절한 시점에 생성되어야 함
- 작업 수행 시간, 중단 시점, 완료 여부가 정확하게 기록되어야 함
- 분석 결과(평균 집중 시간, 수행 패턴)가 실제 데이터 기반으로 정확해야 함
### (1)-2. 신뢰성
- 데이터 기록이 안정적으로 이루어져야 함
- 장시간 이용 시에도 데이터 손실 및 오류 없이 지속적으로 동작해야 함
### (1)-3. 성능
- 일정 생성, 기록, 분석 결과 제공이 즉각적으로 이루어져야 함
- 응답 지연 시 사용자 집중 흐름이 깨질 우려가 있음
### (1)-4. 사용자 친숙성
- 복잡한 학습 없이 사용할 수 있도록 사용자 중심의 인터페이스가 구현되어야 함
- 분석 결과를 직관적으로 이해할 수 있도록 시각화가 필요함
### (1)-5. 가용성
- 서버 장애 등 없이 지속적으로 운영되어야 함.
- 사용자가 필요한 경우 항상 기록 및 확인이 가능해야 함
### (1)-6. 견고성
- 비정상 입력에도 시스템이 오류 없이 동작해야 함 (예외상황에 대한 대비)
## (2) 개발자 혹은 유지보수 담당자가 원하는 기대치는 무엇인가?
### (2)-1. 유지 보수성
- 일정 생성 로직 및 분석 알고리즘 수정 시 유연해야 함
- 기능 확장 및 개선(AI를 활용한 확인 도입 등)에 대응할 수 있어야 함
### (2)-2. 검증 가능성
- 분석 결과가 어떤 데이터와 기준을 기반으로 도출되었는지 확인 가능해야 함
- 동일한 데이터 입력 시 동일한 결과가 나와야 함
### (2)-3. 이식성
- 다양한 환경에서 안정적으로 동작해야 함 (추후 확장 시)
### (2)-4. 재사용성
- 로직을 설계할 시 다른 기능에서도 활용가능하도록 해야 함
### (2)-5. 상호 운용성
- 향후 외부 서비스(외부 캘린터 및 알림 등)와 연동 가능해야 함
## (3) 개발 조직의 관리자가 원하는 기대치는 무엇인가?
- 재사용 가능한 구조를 통한 개발 비용 절감
- 적은 자원으로 안정적인 서비스 운영 (성능 최적화)
- 명확한 시스템 구조 설계로 유지 보수 및 개선 용이
- 기능 확장에 대응 가능한 구조
- 사용자 증가에 대응 가능한 구조
- 행동 데이터가 정확하게 기록되고 유지되는 구조 (데이터 신뢰성 확보)

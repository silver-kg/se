## 1. 개요

### 1.1 문서의 목적

본 문서는 미루기 방지 및 사용자 업무 관리 시스템의 개발을 위한 요구사항을 정의한다. 사용자의 요구사항과 개발 관련 요구사항을 체계적으로 기술함으로써, 개발팀과 이해관계자 간의 명확한 의사소통 기준을 제공하는 것을 목적으로 한다.

### 1.2 프로젝트 개요

#### 1.2.1 배경 및 필요성

많은 사용자들이 작업을 등록하고도 실제 마감일 직전까지 미루는 행동 패턴을 보인다. 이를 해결하기 위해, 실제 마감일보다 앞선 가짜 마감 기한을 자동으로 생성하고, 사용자의 행동 데이터를 기록하여 스스로의 패턴을 인식하도록 돕는 시스템이 필요하다.

#### 1.2.2 프로젝트 목표

- 사용자의 미루기 습관을 효과적으로 개선하는 시스템을 구현한다.
- 단순한 일정 관리가 아닌, 행동 유도와 자기 인식을 동시에 지원한다.
- 직관적이고 사용하기 쉬운 UI를 제공한다.
- 정확한 행동 데이터 수집 및 분석을 통한 개인화된 인사이트를 제공한다.

#### 1.2.3 사용자 기대치 및 기능적 요구사항

1. **미루기 방지:** 사용자가 "다음 주 금요일까지 보고서 작성"이라는 작업을 등록하면, 시스템이 자동으로 "다음 주 수요일"을 가짜 마감으로 설정하여 여유 있게 작업을 완료할 수 있도록 유도한다.
2. **행동 패턴 인식:** "나는 주로 오후 2~5시에 집중을 잘하는구나", "평균적으로 30분 집중 후 10분 휴식을 취하는구나" 등 자신의 작업 패턴을 객관적으로 확인할 수 있다.
3. **현실적 계획 수립:** "이전에 비슷한 작업에 평균 3시간 걸렸으니, 이번에도 3~4시간 정도 배정해야겠다"는 식의 데이터 기반 계획이 가능하다.

## 2. 요구사항

### 2.1 기능적 요구사항

| 식별자(ID) | 그룹(분류) | 요구사항 상세 내용 | 우선순위 |
|------------|-----------|-------------------|----------|
| FR-001 | 작업 관리 | 사용자는 작업명, 실제 마감일, 예상 소요 시간을 입력하여 새로운 작업을 등록한다. | High |
| FR-002 | 가짜 마감 생성 | 시스템은 실제 마감일로부터 예상 소요 시간의 150%를 역산하여 가짜 마감일을 계산한다. 단, 최소 1일 이상 여유를 보장하여 표시한다. | High |
| FR-003 | 가짜 마감 생성 | 사용자는 시스템이 생성한 가짜 마감일을 수동으로 조정한다. | Medium |
| FR-004 | 작업 수행 기록 | 사용자는 작업 시작/일시정지/재시작/완료 버튼을 통해 작업 수행을 기록한다. 시스템은 각 상태 전환 시각과 누적 수행 시간을 초 단위로 기록한다.| High |
| FR-005 | 작업 수행 기록 | 시스템은 각 작업의 시작부터 일시정지 또는 완료까지의 경과 시간을 누적하여 저장하며, 앱 종료 시 현재까지의 수행 시간을 자동으로 저장한다. | High |
| FR-006 | 패턴 분석 | 시스템은 0~6시(새벽), 6~12시(오전), 12~18시(오후), 18~24시(저녁) 4개 시간대로 구분하여 집중도 통계를 그래프로 시각화하여 제공한다. | Medium |
| FR-007 | 패턴 분석 | 사용자는 "계획한 시간 vs 실제 소요 시간" 비교 차트를 작업별로 확인한다. | Medium |
| FR-008 | 패턴 분석 | 시스템은 작업명에 포함된 키워드 또는 작업의 카테고리를 기준으로 과거 유사 작업의 소요시간을 분석하여 새 작업 등록 시 새로운 작업의 예상 소요 시간을 추천한다. (단, 유사 작업이 3건 이상 존재할 경우에 동작한다.) | Low |
| FR-009 | 리포트 | 사용자는 주간/월간 단위로 작업 완료율, 평균 집중 시간, 가장 생산적인 시간대 등의 리포트를 조회한다. | Medium |
| FR-010 | 알림 | 시스템은 가짜 마감일 D-3, D-1, D-Day에 푸시 알림을 띄운다. | High |

## 3. 품질 요구사항

### 3.1 정확성

- 시스템은 가짜 마감 기한을 실제 마감일보다 최소 1일 이상 앞선 시점에 생성한다.
- 시스템은 작업 수행 시간, 중단 시점, 완료 여부를 오차 없이 기록한다.
- 시스템은 분석 결과(평균 집중 시간, 수행 패턴)를 기존 데이터에 기반하여 산출한다.

### 3.2 신뢰성 (Reliability)

- 시스템은 앱이 비정상 종료되더라도 마지막 저장 시점의 작업 수행 데이터를 복구할 수 있어야 한다.
- 시스템은 네트워크 연결이 없는 오프라인 상태에서도 작업 등록 및 타이머 기능을 정상적으로 제공한다.

### 3.3 성능

- 시스템은 1,000개 이상의 작업 기록이 누적된 상태에서도 리포트 화면의 응답 시간이 3초를 초과하지 않아야 한다.
- 시스템은 작업 등록, 기록, 분석 결과 제공에 대한 응답을 1초 이내로 응답한다.

## 5. 기타 고려 사항

- **개인정보 보호:** 수집되는 행동 데이터는 사용자 동의 하에 수집되며, 관련 법령(개인정보 보호법)을 준수한다.
- **접근성:** 시스템은 시각 장애 사용자를 위해 접근성 서비스를 지원한다.
- **확장성:** 향후 iOS 버전 및 웹 대시보드 개발을 고려하여 백엔드 API를 특정 플랫폼에 종속되지 않는 범용적인 구조로 설계할 수 있도록 한다.

## 1. 서론
### 1.1 목적 및 범위
본 문서는 미루기 방지 및 사용자 업무 관리 시스템의 객체지향 분석 결과를 정의한다. 유스 케이스 다이어그램, 클래스 다이어그램, 순차 다이어그램을 통해 시스템의 기능적 요구사항과 구조적 설계를 명세하며, 개발팀과 이해관계자 간의 명확한 의사소통 기준을 제공한다.

### 1.2 용어 정의
| 용어 | 설명 |
|-----------|------|
| Room DB | Android 로컬 DB (SQLite 기반) |
| FCM | Firebase Cloud Messaging (푸시 알림 서비스) |

## 2. 시스템 개요
### 2.1 소프트웨어 컨텍스트(Context)
#### 2.1.1 Actor Table
| Actor | Role |
|-----------|------|
| 사용자 | 작업을 등록하고 타이머를 통해 작업 수행을 기록하며, 패턴 분석 리포트를 확인하는 주체 |
| 시스템 | 가짜 마감일 생성, 알림 전송, 데이터 분석 등을 자동으로 수행하는 주체 |
